HTML

HTML Boilerplate

It is the basic structure or boilerplate that needs to be set up before anything useful can be done

DOCTYPE

It is very important and tells the browser it's looking at an html file. It also informs the browser what version of HTML it should use to render the document.

<!DOCTYPE html>

HTML Tag

This is the root element of the document. Meaning that every other element of the document is a descendant of it.

<html> </html>

Head Element

Head is where the extra information goes it does not show on the screen but it is essential in getting things to work properly.

<head> </head>

Title Element

This shows up in the tab; it is the title that shows up in the google search. It is always put as a descendant/child of the head element.

<title> I am a title that will show in the browser tab. </title>

Charset Meta Element

Tells the browser what character set (or kinda like what type of alphabet) you are using for the page. It is also always put as a descendant/child of the head element.

<meta charset="utf-8">

Body Element

Body is where all the information goes. Everything you want to show up on the screen or your page goes here.

<body> </body>

HTML Texts

Paragraphs

Paragraphs has <p> and </p> as opening and closing tags, respectively.

If you want to make two paragraphs, you have to put each paragraphs in separate tags with their own opening and closing paragraph tags.

Example:

<p> This is my first paragraph. </p>

<p> This is my second paragraph. </p>

Which will render in your page as:

This is my first paragraph.

This is my second paragraph.

Instead of:

This is my first paragraph.This is my second paragraph.

When you code it under one paragraph tag pair.

i.e.

<p> This is my first paragraph.

This is my second paragraph. </p>

Headings are different than texts in HTML. Headings are written in bold with different sizes.

There are six different headings that varies in importance. h1 is the highest level and is therefore the biggest and has higher importance. h6 is the lowest level which is therefore the smallest in the headings.

<h1> Heading 1 </h1>

<h2> Heading 2 </h2>

<h3> Heading 3 </h3>

<h4> Heading 4 </h4>

<h5> Heading 5 </h5>

Strong Element

It is used for making texts bold.

Sometimes, when googling how to make texts bold in html, you might see answers to use <b> </b>. It does make texts bold visually, the difference is, with web design, we have to give emphasis or importance to some texts for accessibility (such as word search in google or text/screen readers) <b> </b> will give the same effect to the text visually but for visually impaired who uses text readers, the text reader will just read it the same way it reads other texts. With <strong>; the tone of voice on the screen reader will change to communicate the importance of the text.

<strong> We are texts wrapped in strong tags to make us bold. </strong>

Emphasis Element

It is used for making texts italic.

Just like with strong element, sometimes, when googling how to make texts italic in html, you will see a different answer which is to use <i> </i> to make the texts visually italic. But as with the strong element, it is better to use the emphasis element instead of its shorter counterpart to give the highlighted text the importance it needs and for the screen reader to communicate it properly to those who uses it.

<em> We are texts wrapped in emphasis tags to make us italic. </em>

<strong> <em> We are italicized bold texts made by the combination of strong and emphasis element </em> </strong>

Nesting and Indentation

We need to indent any elements that are within other elements. This is known as nesting elements.

When we nest elements within other elements, we create a parent and child relationship between them. The nested elements are the children and the element they are nested within is the parent.

For example in the following example the body element is the parent and the paragraph is the child.

<html>

<head>

</head>

<body>

<p> We are an example. </p>

</body>

</html>

Lists

Unordered Lists

When you need a list of items but the order does not matter.

Unordered lists are created using <ul> element and each item within the list is created using <li>.

In an unordered list, each item starts with a bullet point.

In code:

<ul>

<li> Item 1 </li>

<li> Item 2 </li>

<li> Item 3 </li>

</ul>

In browser:

  • Item 1
  • Item 2
  • Item 3

Ordered Lists

Used when the order of the list of items does matter.

Unordered lists are created using <ol> element and each item within the list is created using <li>. However, each item starts with a number.

In code:

<ol>

<li> Item 1 </li>

<li> Item 2 </li>

<li> Item 3 </li>

</ol>

In browser:

  1. Item 1
  2. Item 2
  3. Item 3

Absolute and Relative Links

Generally, there are two kinds of links we will create:

  1. Links to pages on other websites on the internet.
  2. Links to pages located on our own websites.

Absolute Links

Links to pages on other websites on the internet are called absolute links. A typical absolute link will be made up of the following parts:

protocol://domain/path

An absolute link will always contain the protocol and domain of the destination.

The hyperlink created earlier under anchor elements topic which leads to the homepage of google is an example of absolute link. It contains the protocol (https:// (?)) and domain (www.google.com).

Relative Links

Links to other pages within our own website are called relative links. Relative links do not include the domain name, since it is another page on the same site, it assumes the domain name will be the same as the page we created the link on. Relative links only include the file path to the other page, relative to the page you are creating the link on.

The previous example where the hyperlink text leads to the homepage is an example of a relative link. The homepage is just within this site and the index.html file which it is in is in the same folder that this filecode is in.

Images

Putting an image in your webpage is kinda like linking sites; you have to put the image's web address in the src (source).

With image sources, if you put the wrong link or file name, it will show up as the broken image icon in your webpage.

The text you put in alt (alternative) attribute is the text that shows up when the image did not load because of wrong link that was put in the src or bad internet connection. It's also for people who uses text reader for those with vision impairment. Text or screen readers reads the text in a page and when it gets to the image, it will read the alt text. Sometime's it is also used by people to cheat systems a little bit for search engine optimization, but if you are doing it correctly, alt should be a description of the image you are linking.

Examples:

In code:

<img src="https://hips.hearstapps.com/hmg-prod.s3.amazonaws.com/images/sunset-quotes-21-1586531574.jpg?crop=1.00xw:0.752xh;0,0.0601xh&resize=768:*" alt="a beautiful sunset by the beach">

In browser:

In code:

<img src="img/hamster.png" alt="cute hamster cartoon that is standing up. it is partly orange and has pink cheek blush.">

In browser:

The first one is an image that has an absolute link. The second one, which is a hamster, is an example of a relative link.
